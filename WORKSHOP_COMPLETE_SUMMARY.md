## Troubleshooting

### Common Issues

**Issue:** Web presentation doesn't load
**Fix:** Check browser console (F12). Ensure using modern browser. Try Chrome.

**Issue:** Speaker notes don't show
**Fix:** Allow pop-ups. Press `P` instead of `S`.

**Issue:** Demos don't work live
**Fix:** Use prepared fallback recordings. Don't panic!

**Issue:** Running over time
**Fix:** Skip interactive exercises. Speed up demos. Extend Q&A less.

**Issue:** Low engagement
**Fix:** Ask more questions. Call on people. Make polls mandatory participation.

**Issue:** Technical questions too advanced
**Fix:** Table complex questions for office hours. Don't derail workshop.

## Tips for Great Delivery

### Before You Present

1. **Practice 3 Times:**
   - First: Alone, with timer
   - Second: With colleague, get feedback
   - Third: In actual room with equipment

2. **Know Your Demos:**
   - Practice each demo 5+ times
   - Have fallback recordings ready
   - Test on presentation computer

3. **Prepare for Questions:**
   - Read all speaker notes
   - Review best practices doc
   - Know where to find answers

### During Presentation

1. **Start Strong:**
   - Open with energy
   - Get audience involved immediately
   - Set clear expectations

2. **Manage Time:**
   - Use timer (press `T`)
   - Check every 10 minutes
   - Be ruthless cutting if over

3. **Engage Audience:**
   - Ask questions frequently
   - Use polls and exercises
   - Call on specific people
   - Encourage chat participation

4. **Handle Questions:**
   - Repeat question for everyone
   - Answer concisely
   - Table complex ones for later
   - Don't let Q&A derail timing

5. **Demo Tips:**
   - Talk through what you're doing
   - Show timer for impact
   - If it fails, use recording
   - Don't debug live

### After Presentation

1. **Collect Feedback:**
   - Send survey immediately
   - Read all comments
   - Identify improvements

2. **Follow Up:**
   - Send materials within 2 hours
   - Answer individual questions
   - Start office hours immediately

3. **Track Adoption:**
   - Monitor Slack channels
   - Check first task completion
   - Measure time saved

4. **Iterate:**
   - Update materials based on feedback
   - Add FAQ document
   - Share lessons learned
